# 스펠링 간격 반복 + 오답 이력 시스템 설계

날짜: 2026-06-10
대상: 영어 단어 모험 (EngStudy)

## 목표

영단어 스펠링 암기력 향상. 현재 스펠링 모드는 무작위 출제·1회 정답 즉시 마스터·오답 미추적 구조라 장기 기억 형성 장치가 없다. 라이트너 박스 기반 간격 반복과 오답 즉시 교정 루프를 도입한다.

## 현재 구조의 공백

1. 간격 반복 없음 — 마스터 여부와 무관하게 모든 단어가 동일 확률로 출제된다.
2. 오답 추적 없음 — 틀린 단어가 어디에도 기록되지 않는다 (`checkSpelling()`은 사운드·카운터만 처리).
3. 교정 학습 없음 — 오답 시 정답 표시만 하고 넘어간다.
4. 마스터 기준 느슨함 — 아무 퀴즈 모드에서든 1회 정답이면 영구 마스터.

## 1. 데이터 모델

`users/{uid}` 문서에 `wordStats` 맵 필드 추가. 키는 `"카테고리키:영단어"` (예: `"family:mother"`).

```javascript
wordStats: {
  "family:mother": {
    box: 3,                // 라이트너 박스 1~5
    dueAt: 1765432100000,  // 다음 복습 기한 (ms timestamp)
    correct: 4,            // 누적 정답 수
    wrong: 1,              // 누적 오답 수
    lastSeen: 1765345700000
  }
}
```

- 단어 360개 기준 약 25KB — Firestore 문서 한도(1MB) 대비 충분히 작아 맵 필드로 충분하다. 서브컬렉션 불필요.
- `store.js`의 `loadState()`/`saveState()`에 `wordStats` 필드만 추가한다.
- 복습 간격: 박스1 = 즉시(0일), 박스2 = 1일, 박스3 = 3일, 박스4 = 7일, 박스5 = 14일.

### 박스 갱신 규칙

- **승급은 스펠링 모드 정답에서만**: box +1 (최대 5), `dueAt = 지금 + 새 박스 간격`. 직접 써야 스펠링 암기로 인정한다.
- 객관식 모드(뜻 맞추기·단어 맞추기)는 `correct`/`wrong` 카운트와 `lastSeen`만 기록한다.
- **오답은 정오답이 있는 퀴즈 모드(스펠링·뜻 맞추기·단어 맞추기)에서** box = 1로 강등, `dueAt = 지금`. 플래시카드·타자 우주는 단어별 정오답 개념이 없으므로 `wordStats`를 갱신하지 않는다.
- 랜덤 카테고리(`isRandom`) 라운드는 기존 `markMastered()`와 동일하게 갱신 제외.
- 커스텀 단어는 `custom:단어` 키로 동일 취급. 단어 삭제 후 남는 고아 stats는 읽을 때 무시한다.

## 2. 출제 우선순위 (스펠링 모드)

`startQuiz('spelling')`의 단어 선택을 무작위 8개에서 우선순위 기반으로 교체한다.

1. 복습 기한 도래 — `dueAt ≤ 지금` (오래 지난 순)
2. 취약 단어 — box 1이면서 오답 이력 있음
3. 새 단어 — 이력 없음
4. 나머지 — 기한 미도래 단어

위 순서로 8칸을 채운 뒤 라운드 내 순서만 셔플한다. 다른 모드는 기존 무작위 유지.

## 3. 오답 즉시 교정 (스펠링 모드)

- **글자 diff 피드백**: 오답 시 정답 단어를 글자 슬롯으로 펼쳐 보여주되, 입력과 같은 위치 글자가 일치하면 초록, 다르면 빨강으로 표시한다. 단순 위치 비교(LCS 불필요).
- **교정 재입력**: 오답이면 "정답을 보고 한 번 따라 써보자!" 입력란이 나타나고, 정답과 정확히 일치해야 `다음 문제` 버튼이 활성화된다. 따라쓰기는 XP·박스에 영향 없음.

## 4. 라운드 끝 오답 재도전

8문제 종료 시 오답이 있으면 `finishQuiz()` 전에 "틀린 단어 다시 도전!" 보너스 라운드가 자동으로 이어진다.

- 틀린 단어만 동일 스펠링 형식으로 재출제, 재도전은 1회만.
- 재도전 정답 XP는 5 (정규 10의 절반). 기존 `comeback` 카운터·`comeback_5` 업적과 연결.
- 재도전 정답은 box 1→2 승급으로 기록.
- 결과 화면에는 본 라운드 + 재도전 합산을 표시한다.

## 5. 홈 화면 "오늘의 복습" 카드

`renderHome()`의 전체 진도 카드 아래에 복습 카드를 추가한다.

- `dueAt ≤ 지금`인 단어 수를 집계해 "🔔 오늘 복습할 단어 N개!" 표시. 0개면 카드 숨김.
- 클릭 시 카테고리 선택을 건너뛰고 복습 전용 스펠링 라운드 시작 — 기한 도래 단어를 카테고리 혼합으로 최대 8개 출제.
- 혼합 라운드의 단어 객체에 출처 카테고리 키를 함께 담아 박스 갱신·`markMastered()`에 사용한다.
- streak·XP는 기존 그대로 적용된다.

## 6. 마스터 기준 변경 (하위 호환)

- 신규 마스터 조건: **박스 4 도달**(= 간격을 두고 스펠링 3회 이상 정답) 시에만 `markMastered()` 호출. 객관식 정답으로는 더 이상 마스터되지 않는다.
- 기존 마스터 단어는 그대로 유지 — `mastered` 배열과 홈 진도 표시는 변경하지 않는다. 마이그레이션 없음.

## 7. 변경 범위

| 파일 | 변경 |
|------|------|
| `js/game.js` | 출제 우선순위, diff 피드백, 교정 재입력, 오답 재도전, 복습 카드, 박스 갱신, 마스터 조건 |
| `js/store.js` | `loadState`/`saveState`에 `wordStats` 추가 |
| `index.html` | diff 슬롯·복습 카드 CSS 소량 추가 |

변경하지 않는 것: 객관식 모드 출제 방식, 업적 시스템 구조, XP/레벨 공식, 기존 `mastered` 데이터.

## 8. 검증 방식

로컬 테스트 없이 `firebase deploy`로 배포 후 실서비스(웹)에서 직접 확인한다.

1. 스펠링 오답 → 글자 diff 표시 + 따라쓰기 후에만 다음 진행.
2. 라운드 종료 → 오답 재도전 라운드 자동 시작, 결과 합산 표시.
3. Firestore `users/{uid}`에 `wordStats` 저장 확인 (콘솔).
4. 복습 카드 — `wordStats`의 `dueAt`을 Firestore 콘솔에서 과거로 수정해 기한 도래 상태를 만든 뒤 홈에 카드가 뜨고 혼합 라운드가 시작되는지 확인.
5. 박스 4 도달 전에는 마스터로 등록되지 않는지 확인.
