# 줌 버튼 FAB 리팩터 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 헤더에서 줌 버튼 2개를 분리해 화면 우상단의 토글 FAB(🔍)로 옮기고, 클릭 시 ➕/➖이 세로 스택으로 펼쳐지게 한다.

**Architecture:** 정적 `<div id="zoomFab">` 컨테이너를 `index.html`에 추가하고, `render()`에서 매번 `renderZoomFab()`로 내부를 갱신. 메뉴 열림 상태(`zoomMenuOpen`)는 모듈 레벨 변수로만 유지(영속화 안 함). 헤더에서 zoom 관련 코드 4줄과 버튼 2개는 제거.

**Tech Stack:** Vanilla JS (ESM), 정적 HTML/CSS. 테스트 프레임워크 없음 — 브라우저에서 수동 검증.

**Spec:** `docs/superpowers/specs/2026-05-11-zoom-fab-design.md`

```js
function renderZoomFab() {
  const triggerClass = zoomMenuOpen ? 'zoom-fab-btn zoom-fab-trigger open' : 'zoom-fab-btn zoom-fab-trigger';
  const zoomInDisabled = zoomLevel >= ZOOM_MAX ? 'disabled' : '';
  const zoomOutDisabled = zoomLevel <= ZOOM_MIN ? 'disabled' : '';
  const actions = zoomMenuOpen
    ? `
      <button class="zoom-fab-btn" onclick="zoomIn()" title="화면 확대" ${zoomInDisabled}>➕</button>
      <button class="zoom-fab-btn" onclick="zoomOut()" title="화면 축소" ${zoomOutDisabled}>➖</button>
    `
    : '';
  return `
    <div class="zoom-fab">
      <button class="${triggerClass}" onclick="toggleZoomMenu(event)" title="화면 크기 조절">🔍</button>
      ${actions}
    </div>
  `;
}
```

- [ ] **Step 2: bootGame에 outside-click 핸들러 등록**

현재 `bootGame` (현재 ~line 1976-1980):

```js
export async function bootGame() {
  await loadState();
  state.screen = 'home';  // welcome 화면 제거 — 이제 항상 home에서 시작
  render();
}
```

다음으로 교체:

```js
export async function bootGame() {
  await loadState();
  document.addEventListener('click', (ev) => {
    if (zoomMenuOpen && !ev.target.closest('#zoomFab')) {
      zoomMenuOpen = false;
      render();
    }
  });
  state.screen = 'home';  // welcome 화면 제거 — 이제 항상 home에서 시작
  render();
}
```

`closest('#zoomFab')`은 클릭이 FAB 컨테이너 안인지 확인 — 컨테이너 안의 모든 자식(트리거, ➕, ➖)에 매치되므로 자체 클릭에는 닫지 않음. 메뉴가 열려 있을 때만 검사하므로 평소엔 성능 영향 없음.

- [ ] **Step 4: 토글 동작**

- 메뉴 열린 상태에서 화면 다른 곳(헤더, 카드, 빈 공간) 클릭 → 메뉴 자동 닫힘.

- [ ] **Step 5: 줌 동작 + 경계**

- 150% 도달 → ➕ disabled (회색).
- ➖ 클릭 → 줄어듦.
- 70% 도달 → ➖ disabled.

- [ ] **Step 6: 영속화**

줌 120%로 맞춘 뒤 메뉴 닫고 페이지 새로고침 → 120%로 시작, 메뉴는 닫힌 상태로.

- [ ] **Step 7: 다른 기능 영향 없음**

게임 모드 진입, 사운드 토글, 업적/어드민 페이지 — 모두 정상. 줌과 메뉴 토글이 다른 기능과 충돌하지 않음.
